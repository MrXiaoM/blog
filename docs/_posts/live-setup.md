近期我打算在B站[直播写代码](http://live.bilibili.com/14365466)，于是顺便在博客分享一下我的直播设备架构。

## 设备拓补

首先是实拍图

![](https://pic1.imgdb.cn/item/691bfa673203f7be00121dd3.webp)  
![](https://pic1.imgdb.cn/item/691bfa673203f7be00121dd2.webp)  

右边第一个无线鼠标是主力电脑的，第二个有线鼠标是推流机的；我的机械键盘可以切换有线和无线 2.4G，有线是主力电脑的，无线是推流机的，但一般推流机用不着键盘，用鼠标就足够开播了。

然后是设备拓补结构图

![](https://pic1.imgdb.cn/item/691bfcd43203f7be00122f96.webp)

其中
+ 采集卡的输入极限是 `1080p 60帧`，价格大概 `70 CNY`，没啥必要就不要录 4K，高清晰度的采集卡很贵。还有就是，无论如何都要将采集卡插到 USB 3.x 上面去，USB 2.0 带宽不够。
+ 推流机是一台低功耗小主机，配置是 `Intel N100`，`16GB 4800Mhz` 板载运行内存，其中还有 `8GB` 运行内存跟核显共享。
+ 摄像头是海康威视的，用于皮套的面部跟踪，它的自带麦克风不好用，所以我用另一套方案来录制我的声音。
+ 旧手机是 Redmi 7A，使用 [Android Mic](https://github.com/teamclouday/AndroidMic) 当做麦克风使用，手机自带麦克风加上软件降噪还是很好使的。
+ 主力电脑是 `AMD R9-6900HX`，`32GB 4800Mhz` 运行内存，核显。

即使是 N100 这种低功耗移动端 CPU 的核显都可以胜任 `1080p 60帧` 的推流工作，只要不是太渣的电脑都可以当做推流机使用。而且这套配置我还能很方便地带走，不管在宿舍还是在家都可以用。

我没钱买高分屏，所以录 1080p 就足够了。有钱买高分屏和显卡，也不会差这几百块买采集卡。再说，你清晰度再高，直播平台和视频平台还不是会给你压码率。

## 软件使用

在推流机上安装了如下软件
+ [OBS Studio](https://obsproject.com/) 用于录制与推流
+ [Android Mic](https://github.com/teamclouday/AndroidMic) + [VB Cable](https://vb-audio.com/Cable/) 用于将手机用作虚拟麦克风
+ [VSeeFace](https://www.vseeface.icu/) 用于渲染皮套
+ 浏览器用于开直播

基本就没有别的软件了，弹幕我是在手机上看的，就放在实拍图左下角那个支架上。

推流机的屏幕太小了，我要监看直播画面以及切换麦克风开关，没地方放弹幕，自然也没去找将弹幕投在 OBS 上的软件。

## 直播策略

我是播着玩的，没有营收目标，只有开发目标。在开播之前想好这次直播要写什么，直播的时候就照常写代码，写完功能并测试完就下播。

直播的时候我会随机播放 lofi 音乐，毕竟钢琴曲太过催眠，我的歌单又会影响我写代码，lofi 有点存在感又不会特别强，最合适不过了。最好是提前找一个好歌单先听一部分，有些歌单虽然排名高，但歌曲风格参差不齐，有些有人声有些没有，有些有标志性鼓组有些没有，听到风格切换就非常烦。
